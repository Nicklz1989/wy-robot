## 版本号管理规范

### 版本号格式
`主版本号.次版本号.修订号` (MAJOR.MINOR.PATCH)

### 变更类型定义

| 类型 | 说明 | 示例 |
|------|------|------|
| MAJOR | 重大架构变更，不兼容的API变更 | V1.0.0 → V2.0.0 |
| MINOR | 新增功能，向后兼容 | V1.0.0 → V1.1.0 |
| PATCH | Bug修复，向后兼容 | V1.0.0 → V1.0.1 |

### 版本发布流程

1. **开发完成**: 代码开发和测试
2. **版本评估**: 确定变更类型
3. **版本标记**: 更新 package.json 版本号
4. **构建测试**: 确保构建通过
5. **部署发布**: 部署到生产环境
6. **版本存档**: 记录变更日志

### 版本回滚流程

如需回滚到 V1.0.0：
```bash
# 查看版本标签
git tag

# 切换到指定版本
git checkout V1.0.0
```
