### 限流配置 (Rate Limit)

系统集成了 Rate Limit 功能，防止 API 滥用和保障服务稳定性：

#### 限流策略
| 策略类型 | 配置值 | 说明 |
|---------|-------|------|
| 单 IP 请求频率 | 30 次/分钟 | 同一 IP 每分钟最多 30 次请求 |
| 单用户请求频率 | 30 次/分钟 | 同一用户每分钟最多 30 次请求 |
| 全局并发限制 | 100 并发 | 系统最大并发请求数 |

#### 实现方法
使用 `@upstash/ratelimit` 库实现分布式限流：

```typescript
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

// 创建限流器
const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(30, "1 m"), // 每分钟 30 次
  analytics: true,
});

// 在 API 路由中使用
export async function POST(request: Request) {
  const ip = request.ip ?? "127.0.0.1";
  const { success, limit, reset, remaining } = await ratelimit.limit(ip);
  
  if (!success) {
    return new Response("请求过于频繁，请稍后再试", { status: 429 });
  }
  
  // 正常处理请求
}
```

#### 技术栈详情
- **前端框架**: Next.js 14 (App Router)
- **部署平台**: Vercel
- **后端服务**: Supabase
- **数据库**: PostgreSQL (Supabase 托管)
- **认证**: Supabase Auth
- **AI 服务**: Moonshot AI (Kimi)
- **限流**: Upstash Redis + Ratelimit

#### 步骤 1: 创建 Supabase 项目

1. 登录 [Supabase Dashboard](https://app.supabase.com)
2. 点击 "New Project"
3. 填写项目名称和密码
4. 等待项目创建完成（约 2-3 分钟）
5. 记录以下连接信息：
   - **Project URL**: `https://<project-ref>.supabase.co`
   - **Anon Key**: 用于客户端的公开密钥
   - **Service Role Key**: 用于服务端的管理密钥

#### 步骤 2: 配置数据库

在 Supabase SQL Editor 中执行以下初始化脚本：

```sql
-- 创建知识库表
CREATE TABLE knowledge_base (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  category VARCHAR(50) NOT NULL,
  title VARCHAR(200) NOT NULL,
  content TEXT NOT NULL,
  version INTEGER DEFAULT 1,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建操作日志表
CREATE TABLE operation_logs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  action VARCHAR(50) NOT NULL,
  details JSONB,
  operator VARCHAR(100),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建管理员表
CREATE TABLE admins (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  username VARCHAR(100) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 插入默认管理员 (密码: Nicklz123)
INSERT INTO admins (username, password_hash) 
VALUES ('Nicklz', '<bcrypt_hash>');

-- 创建索引
CREATE INDEX idx_knowledge_category ON knowledge_base(category);
CREATE INDEX idx_logs_created_at ON operation_logs(created_at);
CREATE INDEX idx_logs_action ON operation_logs(action);
```

#### 步骤 3: 创建 Vercel 项目

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 "Add New Project"
3. 导入 Git 仓库或选择本地项目
4. 配置项目：
   - **Framework Preset**: Next.js
   - **Root Directory**: `./` (或项目子目录)

#### 步骤 4: 配置环境变量

在 Vercel 项目设置的 "Environment Variables" 中添加：

| 变量名 | 值 | 说明 |
|-------|-----|------|

| `UPSTASH_REDIS_REST_URL` | `https://<your-url>.upstash.io` | Upstash Redis URL |

在 Vercel 项目设置的 "Build & Development Settings" 中：

#### 步骤 6: 部署项目

4. 等待 DNS 生效（通常 5-60 分钟）
5. 配置 SSL 证书（Vercel 自动提供）

#### 步骤 8: 验证部署

1. 访问用户端：`https://your-domain.com`
